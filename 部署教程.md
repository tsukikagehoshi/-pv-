# 部署教程

本工具分两种运行方式，按需选择：

| 方式 | 能用什么 | 需要什么 |
|---|---|---|
| **纯静态** | 播放器 + PV 制作 + 导出视频，**但不能粘链接解析歌词** | 一个浏览器（或任意静态托管） |
| **带解析** | 上面全部 + **网易云链接自动解析歌词** | Node.js 18+ 运行 `server.js` |

> 歌词你也可以手动上传 `.lrc` 或粘贴文本，那样即使纯静态也不受影响。链接解析只是省事的附加功能。

下面分「本地」和「云端」两部分。

---

## 一、本地部署

### 方式 A：纯静态（最简单，零安装）
1. 下载本项目（`Code → Download ZIP` 或 `git clone`）。
2. 解压后 **双击 `index.html`**，浏览器直接打开即用。
3. 在页面里上传音频、封面、歌词，开始制作。

> 注意：双击打开时地址栏是 `file:///...`，此时**无法**使用「粘贴网易云链接解析」（浏览器跨域限制）。要用解析请看方式 B。

### 方式 B：本地带解析（需要 Node.js）
1. 安装 [Node.js 18+](https://nodejs.org/)（装完在终端执行 `node -v` 应显示版本号）。
2. 在项目目录打开终端，依次执行：
   ```bash
   npm install
   npm start
   ```
3. 终端显示 `▶ 播放器 / PV 工具已启动：http://localhost:8080` 后，
   浏览器访问 **`http://localhost:8080`**（注意是这个地址，不是双击打开的 file 路径）。
4. 现在「粘贴网易云链接解析歌词」可用了。

> ⚠️ 常见错误：如果解析报「Failed to fetch」，几乎都是因为你用双击 `index.html`（file:// 地址）打开的。必须通过 `http://localhost:8080` 访问。

停止服务：终端按 `Ctrl + C`。

## 二、云端部署

### 方案一：纯静态托管（推荐给不需要解析的人，免费省心）
把 `index.html`、`styles.css`、`app.js` 三个文件（连同其它文件一起也行）传上去即可：

- **GitHub Pages**：仓库 → Settings → Pages → Source 选 `main` 分支 → 保存，几分钟后得到 `https://你的用户名.github.io/仓库名/`。
- **Vercel / Netlify**：导入 GitHub 仓库，框架选「Other / 无」，构建命令留空，发布目录填根目录，部署即可。

这种方式**不能用链接解析歌词**（没有后端），但播放器、PV、导出全部可用。

### 方案二：云服务器跑 server.js（功能全，含解析）
以 **腾讯云轻量应用服务器** 为例（其它云类似），适配 2 核 2GB、Ubuntu。

**1. 开放端口**
腾讯云控制台 → 你的实例 → 防火墙，放通：
| 协议端口 | 用途 |
|---|---|
| TCP:22 | SSH 登录 |
| TCP:80 / 443 | 网页访问 / HTTPS |
| TCP:8080 | Node 服务（配了 Nginx 后可关） |

**2. 连上服务器并装 Node.js**
```bash
ssh root@你的公网IP
sudo apt update && sudo apt upgrade -y
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs git
node -v   # 确认 v20.x
```

**3. 拉代码、装依赖、启动**
```bash
cd /opt
git clone https://github.com/你的用户名/你的仓库名.git yunyin
cd yunyin
npm install --production
```

**4. 用 PM2 常驻运行（关掉终端也不停）**
```bash
sudo npm install -g pm2
pm2 start server.js --name yunyin
pm2 startup        # 按提示再执行它输出的那条命令
pm2 save
```
此时访问 `http://你的公网IP:8080` 即可使用（含解析）。
常用命令：`pm2 status` / `pm2 logs yunyin` / `pm2 restart yunyin` / `pm2 stop yunyin`。

**5.（可选）配 Nginx + 域名 + HTTPS**
```bash
sudo apt install -y nginx
sudo vim /etc/nginx/sites-available/yunyin
```
写入（把域名换成你的）：
```nginx
server {
    listen 80;
    server_name pv.你的域名.com;
    client_max_body_size 50m;
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```
```bash
sudo ln -s /etc/nginx/sites-available/yunyin /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
# 免费 HTTPS
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d pv.你的域名.com
```
完成后访问 `https://pv.你的域名.com`。

> **境外服务器提示**：若服务器在海外（如首尔），访问网易云接口可能偶发超时/限流，属网络链路问题，不影响手动上传歌词。境外 IP 通常**无需 ICP 备案**。

## 更新代码
```bash
cd /opt/yunyin
git pull
npm install --production   # 依赖有变动才需要
pm2 restart yunyin         # 纯静态方案无需重启，刷新网页即可
```

## 关于配置与性能
- 真正的渲染（Canvas 绘制、视频录制）都在**访问者自己的浏览器**里完成，服务器只做静态托管与歌词转发，几乎不吃资源，2 核 2GB 足够个人使用。
- 导出的 `.webm` 想转 `mp4`：本地用 `ffmpeg -i pv.webm pv.mp4`。
